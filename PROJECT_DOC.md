# 测试开发面试速成站项目文档

## 项目目标

构建一个面向测试开发岗位面试准备的中文内容站，帮助用户在较短时间内建立可用于面试表达的知识骨架。当前生产实现基于 Astro + Starlight，目标是在保证静态站点性能和可维护性的前提下，持续扩展高频面试内容。

站点核心覆盖：

- 术语体系
- 技术专题
- 项目类型
- 场景题
- 编码题
- 学习路线
- AI 时代成长指南
- 练手模板库
- 面试追问链

## 产品定位

- 面向对象：测试开发初学者、转岗用户、准备中大厂测试开发面试的候选人
- 产品类型：测试开发面试速记站
- 价值主张：用结构化内容帮助用户快速补齐知识框架，并能在面试中说清楚概念、项目和方案

## 设计方向

整体视觉参考现代文档站的阅读体验与信息架构，但不复刻现成品牌。页面风格强调：

- 文档型清晰层级
- 克制配色
- 强导航感
- 高可读性
- 轻边框、弱装饰、少卡片

首页使用“速成地图”结构，突出高频入口和学习路径。内容页依托 Starlight 提供稳定的文档壳层，并在其上扩展搜索、进度、收藏、自测题和分享等交互。

## 信息架构

### 首页

- 价值主张
- 3 天 / 7 天学习路线入口
- 各核心模块推荐入口
- 最近浏览与学习进度

### 分类与内容

当前分类包括：

- glossary
- tech
- project
- scenario
- coding
- roadmap
- ai-learning
- practice-template
- interview-chains

每个分类内容以 Markdown 文件存放在 `src/content/docs/<category>/` 下，由 Starlight 自动生成分类页与详情页。

## 关键交互

### 术语悬浮解释

- 正文中的术语可显示悬浮解释
- 桌面端支持 hover，点击可保持显示
- 移动端支持点击打开
- 悬浮层展示简要定义和详情链接

### 搜索与筛选

- 使用 Starlight / Pagefind 提供全站搜索
- 补充标签页与难度页作为快速索引入口

### 学习进度

- 使用 localStorage 记录学习完成状态
- 记录最近浏览内容
- 不引入账号系统

### 自测题与模板复用

- 内容页支持内嵌自测题
- 模板页提供结构化练手模板和表达模板
- 支持本地收藏与最近查看

## 技术方案

- 框架：Astro 4
- 文档壳层：`@astrojs/starlight`
- 语言：TypeScript
- 内容管理：仓库内 Markdown + schema 校验
- 搜索：Pagefind（由 Starlight 构建流程产出）
- 状态持久化：localStorage
- 部署：GitHub Pages

## 目录约定

- `src/pages/` - 首页与额外索引页
- `src/components/` - Astro 组件
- `src/lib/` - 客户端和页面辅助逻辑
- `src/content/config.ts` - 内容 schema
- `src/content/docs/` - 全部站点内容
- `tests/` - 单元、a11y、e2e 测试
- `scripts/` - 内容迁移辅助脚本
- `temp-site/` - 历史 Next.js 迁移来源，不属于当前运行时

## 工程规则

- 首页推荐链接和分类推荐必须来自统一配置，不允许继续手写分散 slug
- 新增内容后应同步更新测试和分类配置
- 发布前必须通过 `npm run check`
- GitHub Pages 工作流需要在构建前执行 lint、typecheck 和 test
