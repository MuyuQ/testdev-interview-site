## 内容质量标准

**术语条目**：定义 + 为什么重要 + 常见误区 + 面试追问 + 答题提示 + 混淆术语

**技术/项目/场景条目**：说明 + 面试怎么问 + 项目里怎么用 + 容易答错什么

**编码条目**：回答要点 + 常见失分点 + 代码思路说明

**模板条目**：目标场景 + 包含内容 + 使用方法 + 扩展建议

**AI 成长条目**：学习目标 + 为什么现在学 + 学习步骤 + 实际用例 + 常见误区 + 面试表达点

## 实施计划

### 分批填充策略

**第一批**：P0 核心高频 54 条，写入 `src/content/data.ts`

**第二批**：P1 重要补充 10 条

**第三批**：P2 扩展内容 5 条

### 技术实现

所有新增内容直接追加到 `src/content/data.ts` 对应数组，复用现有类型定义：
- `GlossaryTerm` 类型用于术语
- `StandardTopic` 类型用于技术、项目、场景、编码、学习路线
- `AILearningGuide` 类型用于 AI 成长
- `PracticeTemplate` 类型用于模板库

无需修改类型系统或组件代码。

### 验证方式

每批填充后：
1. 运行 `npm run build` 确认无类型错误
2. 运行 `npm run dev` 本地预览
3. 验证首页、分类页、详情页正确加载新内容
