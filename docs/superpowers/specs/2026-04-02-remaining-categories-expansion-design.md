# 剩余分类内容深度扩展设计文档

## 项目背景

测试开发面试速成站已完成 Tech、Scenario、Coding 三大分类的内容深度扩展（20 个条目，+3720 行）。现需要继续扩展剩余 4 个分类的内容深度，使全站内容对初学者友好。

## 需求总结

| 分类 | 条目数 | 模板类型 | 扩展方式 |
|------|--------|---------|---------|
| Glossary | 20 | 统一 5 模块 | 并行执行 |
| Project | 12 | 项目专用 4 模块 | 并行执行 |
| Roadmap | 8 | 路线专用 4 模块 | 并行执行 |
| AI-Learning | 8 | AI 学习专用 4 模块 | 并行执行 |

**总条目数**：48 个

## 扩展模板设计

### Glossary 分类扩展模板（统一 5 模块）

每个 Glossary 条目扩展为以下 5 个模块：

#### 模块 1：基础入门

**内容要求**：
- 是什么：300-500 中文字详细解释术语定义、核心概念
- 为什么重要：该术语在测试开发中的重要性、面试高频原因
- 相关术语对比：与易混淆术语的对比表格
- 前置知识：必须掌握、建议掌握、不需要掌握

**示例结构**：
```
## 基础入门：[术语名称] 是什么 & 为什么重要

### [术语名称] 是什么
（300-500 中文字详细解释）

### 为什么测试开发要理解 [术语名称]
（5 个理由，每个 100-200 中文字）

### 相关术语对比
| 术语 | 区别 |
|------|------|
| 相关术语 A | 区别说明 |
| 相关术语 B | 区别说明 |

### 前置知识
（必须掌握、建议掌握、不需要掌握）
```

#### 模块 2：学习路径

**内容要求**：
- 零基础第一步：10 分钟快速理解该术语的体验
- 学习阶段：理解概念 → 实践应用 → 面试表达
- 时间建议：不同投入的学习周期

**示例结构**：
```
## 学习路径：从理解到能面试表达

### 零基础第一步
（10 分钟快速理解任务）

### 学习阶段
- 阶段 1：理解概念（时间、目标、内容）
- 阶段 2：实践应用（时间、目标、内容）
- 阶段 3：面试表达（时间、目标、内容）

### 时间建议
（不同时间投入的学习周期）
```

#### 模块 3：实操案例

**内容要求**：
- 案例 1：在项目中的应用（场景描述、应用流程、关键要点）
- 案例 2：常见使用场景（不同项目类型中的应用）

**示例结构**：
```
## 实操案例：在项目中如何应用

### 案例 1：[场景名称]
- 场景描述
- 应用流程（用流程描述，不用完整代码）
- 关键要点

### 案例 2：[场景名称]
- 场景描述
- 应用流程
- 关键要点
```

#### 模块 4：常见误区（5 个）

**内容要求**：
每个误区包含：
- 错误理解：初学者常见的错误认知
- 正确理解：准确的概念解释
- 面试应对：如何在面试中正确表达

**示例结构**：
```
## 常见误区：初学者最容易理解错的地方

### 误区 1：[错误理解概述]
- 错误理解：具体描述
- 正确理解：准确解释
- 面试应对：如何正确表达

### 误区 2-5：（同理）

### 误区总结
（对照表）
```

#### 模块 5：面试问答（5 个问题）

**内容要求**：
每个问题包含：
- 回答骨架：30 秒内讲完核心
- 深度答案：承接追问
- 追问应对：2-3 个常见追问
- 优先级标注：P0（必问）/ P1（常问）

**示例结构**：
```
## 面试问答：如何把概念讲清楚

### Q1：[面试常见问题] 【P0 高频】
- 回答骨架：核心观点
- 深度答案：展开解释
- 追问应对：
  - 追问 1：应对策略
  - 追问 2：应对策略

### Q2-5：（同理）

### 面试回答核心技巧
（技巧表格）
```

### Project 分类扩展模板（项目专用 4 模块）

每个 Project 条目扩展为以下 4 个模块：

**内容要求**：
- 项目概述：项目类型、业务领域、用户群体

```
- 架构特点
- 业务复杂度

```

**内容要求**：

- 关键业务场景：高频业务场景

### 并行执行策略

**执行方式**：
- 48 个条目全部并行执行
- 每个条目由一个独立的 subagent 完成
- 根据分类使用对应的扩展模板

**Agent Prompt 结构**：
```
## 任务：扩展 [条目名称] 条目

**文件位置**：`temp-site/src/content/data.ts` 中 slug: "[slug]" 的条目

**扩展模板**：按照 [分类名称] 分类扩展模板（[X] 个模块）

### 模块 1：[模块名称]
（具体要求）

### 模块 2：[模块名称]
（具体要求）

...

**要求**：
1. 保持现有结构
2. 内容用中文
3. 流程描述代替完整代码
4. 完成后 git commit: `feat(content): expand [slug] [category] with beginner-friendly content`

**输出**：返回扩展摘要
```

## 质量标准

### 内容深度标准
- Glossary：每个条目 1500-2500 中文字
- Project：每个条目 1500-2000 中文字
- Roadmap：每个条目 1000-1500 中文字
- AI-Learning：每个条目 1200-1800 中文字

### 初学者适配标准
- 所有代码逻辑用流程描述
- 术语首次出现时有解释
- 有"零基础第一步"引导
- 有面试回答骨架

### 验收标准
- 流程描述清晰，无需看代码即可理解
- 面试回答可以直接用于准备
- 初学者看完能知道"是什么、怎么学、怎么用、怎么答"
