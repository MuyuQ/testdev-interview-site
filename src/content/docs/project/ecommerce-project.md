## 业务流程

- 商品浏览与搜索。
- 购物车、下单、库存锁定与支付。
- 履约发货、售后退款与营销核销。

## 风险点

- 库存与订单状态不一致。
- 优惠叠加、价格计算和边界金额错误。
- 高并发活动下超卖、重复下单和回调延迟。

## 测试策略

- 核心链路做接口回归，前台关键页面做少量 UI 回归。
- 营销和价格相关规则做数据驱动组合测试。
- 下单支付链路增加日志与数据库双重校验。

## 可讲成果

- 核心下单回归从人工 3 小时缩到 25 分钟。
- 通过门禁阻断多次高风险价格配置上线。
- 补齐库存和支付异常链路后，线上重复单率下降。
