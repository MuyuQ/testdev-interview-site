## 路线概述

- 适用人群：准备测试开发岗位面试的求职者，特别是面试前 1-3 天需要快速检查的准备者。
- 学习目标：系统梳理面试准备事项，确保知识点、表达、心态和物品四大维度无遗漏。
- 前置条件：已完成基础学习，有一定项目经验，准备进入面试冲刺阶段。
- 时间投入：首次完整检查约 2-3 小时，后续快速检查约 30 分钟。

## 每日安排

- 面试前 3 天：完成知识点自查，标记薄弱项，重点复习高频问题。
- 面试前 2 天：完成项目表达演练，录制自我介绍和项目介绍视频并复盘。
- 面试前 1 天：完成物品清单准备，打印简历、准备作品集、检查在线作品链接。
- 面试当天：提前 30 分钟检查设备（远程面试）或路线（现场面试），进行热身表达练习。

### 面试前 3 天：知识点自查

**技术基础检查**：

- [ ] HTTP 协议：状态码、请求方法、Header、Cookie/Session
- [ ] 数据库：SQL 查询、索引原理、事务隔离级别
- [ ] 缓存：Redis 数据类型、缓存策略、缓存穿透/击穿/雪崩
- [ ] 消息队列：Kafka/RabbitMQ 基本概念、消息可靠性
- [ ] 测试理论：测试用例设计方法、测试金字塔、Bug 生命周期

**测试专业技能检查**：

- [ ] 接口测试：测试方法、工具使用、断言策略
- [ ] 自动化框架：Pytest/UnitTest 核心概念、Fixture、参数化
- [ ] UI 自动化：Selenium/Appium 原理、定位策略、等待机制
- [ ] 性能测试：指标理解、压测工具、瓶颈分析
- [ ] CI/CD：流水线搭建、质量门禁、自动化集成

**场景题检查**：

- [ ] 支付回调：流程理解、幂等设计、异常处理
- [ ] 登录鉴权：Token 机制、权限验证、安全测试
- [ ] 数据一致性：分布式事务、最终一致性、对账机制
- [ ] 高并发场景：限流、降级、熔断、库存扣减

### 面试前 2 天：表达演练

**自我介绍演练**：

- [ ] 1 分钟版本流畅自然
- [ ] 3 分钟版本结构完整
- [ ] 录音回听，检查语速和口头禅

**项目介绍演练**：

- [ ] 核心项目 1：背景-问题-方案-收益-反思，3 分钟内讲完
- [ ] 核心项目 2：同上，准备 3 个可能的追问
- [ ] 每个项目都有具体数据支撑

**技术问题表达**：

- [ ] 练习"先说结论再展开"的表达方式
- [ ] 每个技术点能举例说明
- [ ] 遇到不会的问题，练习诚实回应话术

### 面试前 1 天：物品准备

**现场面试**：

- [ ] 打印 3 份简历（面试官可能多于一人）
- [ ] 身份证、学历证等必要证件
- [ ] 笔和笔记本（记录问题）
- [ ] 作品集或技术文档（如有）
- [ ] 提前查好路线，预留 30 分钟缓冲

**远程面试**：

- [ ] 摄像头、麦克风测试正常
- [ ] 网络稳定，准备手机热点备用
- [ ] 面试软件安装并登录（腾讯会议、飞书等）
- [ ] 背景整洁，光线充足
- [ ] 简历和项目资料放在屏幕旁边便于参考

### 面试当天

**提前准备**：

- [ ] 提前 30 分钟到达/上线
- [ ] 做 3 次深呼吸，缓解紧张
- [ ] 口头热身：大声说一遍自我介绍
- [ ] 检查仪容仪表

**面试中**：

- [ ] 保持眼神交流（远程面试看摄像头）
- [ ] 语速适中，每讲完一个点停顿 1-2 秒
- [ ] 遇到不会的问题，诚实承认 + 关联经验 + 学习计划
- [ ] 最后提问环节准备 2-3 个有深度的问题

## 学习方法

- 清单打勾法：打印清单逐项检查，完成后打勾，避免遗漏。
- 模拟测试法：找朋友或 AI 进行模拟面试，验证准备效果。
- 录音复盘法：录下自己的回答，检查语速、逻辑和表达是否清晰。
- 错题本法：把每次模拟中答不好的问题记录下来，专项突破。
- 常见误区：只准备知识点不练表达、只看不说不输出、准备过多导致记忆混乱。

## 面试表达

- 自我介绍：准备 1 分钟简短版和 3 分钟详细版，涵盖背景、项目、技能和求职意向。
- 项目表达：按背景-问题-方案-收益-反思五段式准备，每个项目准备 2-3 分钟讲述。
- 技术问题：先说结论再展开细节，用数据支撑观点，承认不知道的部分并说明学习计划。
- 行为问题：准备团队协作、冲突处理、成长经历各 1-2 个案例，用 STAR 法则讲述。
- 提问环节：准备 3-5 个有深度的问题，展示对岗位和公司的了解。

### 提问环节建议问题

**关于团队**：

- "测试团队目前的质量保障体系是怎样的？"
- "团队目前面临的最大质量挑战是什么？"

**关于技术**：

- "团队目前自动化测试的覆盖率如何？主要用什么框架？"
- "CI/CD 流程中质量门禁是怎么设计的？"

**关于成长**：

- "这个岗位的晋升路径是怎样的？"
- "团队有技术分享或学习机制吗？"

**避免问的问题**：

- 薪资福利（等 HR 主动提或后续沟通）
- 加班情况（可以换一种方式问："团队的工作节奏是怎样的？"）
- 网上能查到的基本信息（显得没做功课）

## 心态管理

### 面试前

- **正常化紧张**：紧张是正常的，说明你在乎这次机会。
- **积极暗示**：回顾自己的成果和经验，建立自信。
- **降低期望**：把面试当成交流机会，不是生死考验。

### 面试中

- **专注当下**：不要想上一个问题答得好不好，专注当前问题。
- **允许停顿**：思考 3-5 秒再回答是正常的，不要急着开口。
- **接受不完美**：没有完美的面试，允许自己有答不好的问题。

### 面试后

- **立即复盘**：记录关键问题和自己的表现。
- **放下结果**：面试结果受多种因素影响，不是对你个人的否定。
- **持续改进**：每次面试都是学习机会，不断优化。
