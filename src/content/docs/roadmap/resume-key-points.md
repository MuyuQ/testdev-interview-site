## 简历结构建议

- 个人信息：姓名、联系方式、求职意向，控制 3 行以内。
- 技能摘要：列出 5-8 项核心技能，与岗位 JD 匹配。
- 工作经历：按时间倒序，每家公司 2-4 条核心产出。
- 项目经历：选择 2-3 个重点项目，突出职责和成果。

### 个人信息

- **必填项**：姓名、手机号、邮箱、求职意向。
- **可选项**：GitHub 链接（有高质量项目时）、博客链接（有技术文章时）。
- **不要写**：照片、年龄、婚姻状况（除非公司明确要求）。
- **格式**：一行或两行排布，节省空间给核心内容。

### 技能摘要

技能摘要是 HR 筛选简历的关键区域，需要精心设计：

**写法原则**：

1. **与 JD 匹配**：研究目标岗位的技能要求，优先写匹配的项。
2. **分级描述**：用"精通"、"熟悉"、"了解"区分掌握程度，诚实描述。
3. **具体化**：不要只写"自动化测试"，写成"熟悉 Pytest 接口自动化框架，有从 0 到 1 搭建经验"。
4. **控制数量**：5-8 项为宜，太多显得杂，太少显得薄。

**示例**：

```
• 精通 Python，熟悉 Java，能独立完成测试框架开发和工具编写
• 熟悉 Pytest + Requests 接口自动化框架，有 200+ 用例维护经验
• 熟悉 CI/CD 流程，能使用 Jenkins/GitLab CI 搭建自动化流水线
• 熟悉 MySQL，能编写复杂 SQL 进行数据验证和测试数据构造
• 了解 Redis、Kafka 等中间件，有相关测试验证经验
• 了解性能测试，能使用 JMeter/Locust 进行接口压测
```

### 工作经历

工作经历展示你的职业发展轨迹，按以下原则撰写：

- **时间倒序**：最近的工作经历放在最前面。
- **每条产出**：用"动作 + 对象 + 结果"的句式，如"主导接口自动化体系建设，覆盖率从 30% 提升到 85%"。
- **量化成果**：用数据支撑，避免空泛描述。
- **突出成长**：展示职责范围扩大、技术能力提升的过程。

### 项目经历

项目经历是技术面面试官重点关注的部分：

**选择标准**：

1. 与你申请的岗位最相关的项目。
2. 你承担核心角色、能讲清楚技术细节的项目。
3. 有可量化成果的项目。

**描述结构**（每个项目 3-4 条）：

- 项目背景：一句话说明项目是什么、解决什么问题。
- 你的职责：明确你在项目中的角色和贡献。
- 技术方案：用了什么技术、为什么这么选。
- 项目成果：用数据说明效果。

## 项目描述要点

- 用 STAR 法则：背景、任务、行动、结果四段描述。
- 量化成果：用百分比、数量、时间等数据支撑。
- 突出个人贡献：避免团队成果和个人贡献混为一谈。
- 技术关键词：埋入面试想被问到的技术点。

### STAR 法则详解

**Situation（背景）**：项目场景、团队规模、业务目标。

> "公司 XX 业务线每次发版需要手动执行 200+ 用例，耗时 2 天。"

**Task（任务）**：你面临的问题和挑战。

> "我的任务是搭建接口自动化体系，缩短回归测试时间。"

**Action（行动）**：你采取的具体行动和技术方案。

> "我基于 Pytest + Requests 设计了自动化框架，封装了数据驱动、断言库和报告模块，集成到 CI/CD 流水线。"

**Result（结果）**：可量化的成果和反思。

> "回归测试时间从 2 天缩短到 2 小时，覆盖率从 30% 提升到 85%，上线后缺陷率降低 40%。"

### 量化成果的技巧

| 维度     | 可量化指标                                        |
| -------- | ------------------------------------------------- |
| 效率提升 | 时间缩短 X%、人力节省 X 人天、执行速度提升 X 倍   |
| 质量提升 | 缺陷率降低 X%、覆盖率提升 X%、漏测率降低 X%       |
| 规模扩大 | 用例数量从 X 到 Y、支持 X 个接口、覆盖 X 个业务线 |
| 成本降低 | 服务器成本降低 X%、工具替代 X 个人工              |

## 常见错误

- 堆砌关键词：写了但被问倒，不如不写。
- 空泛描述："参与测试" 无法体现价值，改成具体产出。
- 格式混乱：字体、间距、对齐保持一致，体现专业性。
- 过时信息：3 年以上的项目一笔带过，聚焦近期成果。

### 错误示例与修正

| 错误写法           | 修正写法                                             |
| ------------------ | ---------------------------------------------------- |
| "参与接口测试"     | "设计并维护 200+ 接口自动化用例，覆盖率 85%"         |
| "负责质量保障"     | "建立质量门禁体系，上线缺陷率降低 40%"               |
| "使用 Pytest 框架" | "基于 Pytest 搭建自动化框架，封装数据驱动和报告模块" |
| "优化测试流程"     | "将回归测试时间从 2 天缩短到 2 小时"                 |

### 简历优化检查清单

提交简历前逐项检查：

- [ ] 所有技能点都能展开讲解 2-3 层
- [ ] 每个项目都有具体数据支撑
- [ ] 没有空泛描述，每条都体现个人价值
- [ ] 格式统一（字体、字号、间距、缩进）
- [ ] 控制在 1-2 页，重点突出
- [ ] 没有错别字和语法错误
- [ ] 联系方式正确有效
- [ ] 技术关键词与目标岗位 JD 匹配
